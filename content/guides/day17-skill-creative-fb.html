Skill Sản Xuất Content
Hoàn Chỉnh + Auto-Post FB

Hôm nay agent của bạn sẽ tự gen full content (ảnh đẹp + văn bản hoàn chỉnh — hook + body + CTA + hashtag) và đăng lên Page Facebook mỗi sáng — không cần bạn động tay. Đồng thời gen được 3 bộ creative ads (ảnh + ad copy ghép cặp) để bạn paste vào Ads Manager khi muốn chạy chiến dịch.

Mục tiêu cuối ngày

Cuối hôm nay bạn có:

Skill tao-creative-fb với 2 mode: (1) Content Free — agent tạo full bài (ảnh + caption hoàn chỉnh) đăng Page mỗi sáng, (2) Creative Ads — agent tạo 3 bộ creative (mỗi bộ gồm ảnh + ad copy) khi bạn muốn chạy chiến dịch Lịch hẹn tự chạy 9h sáng mỗi ngày — gen ý tưởng → bạn chọn → agent làm full content → bạn duyệt → đăng tự động Lần đầu trải nghiệm: dậy sáng thấy Page Facebook đã có 1 bài đăng đầy đủ ảnh + văn bản, không phải bạn làm
Tại sao đây là moment chuyển hoá

Trước hôm nay bạn (hoặc team bạn) phải vừa nghĩ ý tưởng, vừa viết bài, vừa làm ảnh, vừa đăng Facebook mỗi ngày — tốn 30-60 phút/ngày. Một content creator junior lương ~8tr/tháng làm việc này.

Sau hôm nay bạn ngủ — agent làm thay. Page có content đều đặn, đúng tone, đẹp visual. Bạn không phải bỏ công, không phải trả lương.

Đọc kỹ trước khi bắt đầu

Hôm nay có setup Facebook API — đây là phần dễ kẹt nhất chuỗi. Nếu bạn vội, bạn sẽ bỏ cuộc giữa chừng.

Cần 1-2 giờ tập trung, không bị ngắt quãng. Lưu kỹ token vào file đúng vị trí (sẽ hướng dẫn). Nếu kẹt — đọc phần Troubleshoot ở mỗi bước, có lời giải cho 90% lỗi phổ biến.

Quy tắc AI-First xuyên suốt ngày hôm nay

Đừng tự loay hoay. Mỗi khi gặp lệnh không hiểu, lỗi không biết sửa, hoặc terminal trông lạ — screenshot + paste vào agent coding (Cursor / Antigravity / Claude Code) và bảo nó: "Giải thích cho tôi" hoặc "Chạy giúp tôi, giải thích output". Agent sẽ làm tất cả.

Bạn không cần là dev. Bạn chỉ cần là người chỉ đạo agent. Đây là pattern cốt lõi của Operator — và là lý do khoá này gọi là AI-First Vibe Coding.

Phần Một
Setup OpenAI + Facebook API

BƯỚC 01 Đăng ký OpenAI API key

Một bài đăng Facebook hoàn chỉnh = ảnh + văn bản (caption). Hôm nay agent của bạn sẽ tự sinh ra cả 2 phần trong cùng 1 lần. Phần ảnh dùng GPT Image API của OpenAI (model gen ảnh tốt nhất hiện tại, text trong ảnh viết chuẩn tiếng Việt, giá rẻ). Phần văn bản dùng GPT-4 hoặc Claude (qua chính agent coding của bạn).

1.1 — Tạo tài khoản OpenAI Platform

  1. Vào https://platform.openai.com (chú ý platform, không phải chatgpt.com).
  2. Đăng ký tài khoản mới (hoặc đăng nhập nếu đã có).
  3. Tài khoản mới được tặng $5 free credit — đủ gen ~500-1000 ảnh chất lượng thường (đủ cho ~1 năm content nếu đăng mỗi ngày).

1.2 — Tạo API key

Settings (icon ⚙️ góc trên phải) → API keys → Create new secret key → đặt tên kp3-content-bot → bấm Create

Một popup hiện ra với chuỗi key bắt đầu bằng sk-proj-.... Copy ngay và lưu vào file Notepad/notes — popup đóng là không xem lại được, phải tạo key mới.

Kiểm tra điểm

Bạn có 1 chuỗi bắt đầu bằng sk-proj-, dài khoảng 100-150 ký tự. Đây là OpenAI API key của bạn.

1.3 — Nạp tiền (nếu hết $5 free)

$5 free đủ làm Ngày 17. Nhưng để dùng lâu dài, có 2 cách nạp:

  • Visa/Mastercard quốc tế (Vietcombank Visa Debit, Techcombank Mastercard...): vào Settings → Billing → Add payment method → nạp tối thiểu $5.
  • Wise / Revolut: nếu thẻ ngân hàng VN bị từ chối (hay xảy ra với account mới), dùng thẻ ảo Wise — nạp VND vào Wise → tạo thẻ ảo USD → add vào OpenAI.
AI-First — kẹt là hỏi agent

Nếu màn hình OpenAI khác hướng dẫn (UI thay đổi liên tục), screenshot màn hình → paste vào agent coding + bảo: "Tôi đang ở đây trên OpenAI Platform, muốn tạo API key, bấm tiếp đâu?". Agent đọc ảnh và chỉ chính xác.

⚠️ Troubleshoot — Bước 01
"Account của tôi bị khoá ngay sau khi đăng ký"

Account VN đôi khi bị flag do IP. Dùng VPN (US/SG) khi đăng ký lần đầu. Sau khi tài khoản được tạo, dùng IP VN bình thường được.

"Thẻ Visa của tôi bị từ chối"

Thử thẻ khác (đặc biệt: thẻ tín dụng tốt hơn thẻ ghi nợ). Nếu vẫn từ chối, dùng Wise — đăng ký 5 phút, nạp VND, tạo thẻ ảo USD. Phí Wise ~10k VND/tháng nhưng chấp nhận khắp nơi.

"Tôi không thấy $5 free credit"

Free credit có thời hạn 3 tháng từ ngày đăng ký. Nếu đăng ký lâu rồi nhưng chưa dùng — credit đã hết hạn. Phải nạp tiền thật để dùng.

BƯỚC 02 Tạo Facebook Developer App

Để code Python đăng bài lên Page Facebook, ta cần 1 Facebook Developer App. App này là "chìa khóa" giúp Facebook nhận biết code của bạn.

Hiểu kiểu đơn giản
Facebook Developer App giống như "thẻ nhân viên" bạn cấp cho code của mình. Có thẻ này, code của bạn được phép vào "tòa nhà Facebook" và làm việc thay bạn — như đăng bài lên Page bạn quản lý.

2.1 — Tạo App tại Facebook for Developers

  1. Vào https://developers.facebook.com — đăng nhập bằng tài khoản Facebook cá nhân của bạn (tài khoản mà bạn đang admin Page kinh doanh).
  2. Nếu chưa từng vào, Facebook sẽ yêu cầu bạn "Register as developer" — đồng ý điều khoản.
  3. Bấm góc trên phải My Apps → Create App.

2.2 — Chọn loại App

Facebook hỏi "What do you want your app to do?". Chọn:

Other → bấm Next → chọn Business → bấm Next
Quan trọng

Phải chọn loại Business. Loại "Consumer" hoặc "Gaming" không có Marketing API. Sai loại = phải tạo lại app từ đầu.

2.3 — Đặt tên + tạo

  1. App name: kp3-content-bot (hoặc gì cũng được, không có chữ "facebook" hay "fb").
  2. Email: email của bạn.
  3. Business Account: chọn Business Manager nếu có; nếu không, để trống cũng được.
  4. Bấm Create App — Facebook hỏi password, nhập password Facebook cá nhân.
Kiểm tra điểm

Bạn được chuyển vào trang Dashboard của App, thấy tên app + một ID dạng 1234567890123456 (15-16 chữ số) ở phía trên. Đây là App ID — sẽ cần ở Bước 03.

2.4 — Lấy App Secret

Sidebar trái → App settings → Basic → tìm dòng App Secret → bấm Show → nhập password Facebook → copy chuỗi

App Secret là chuỗi 32 ký tự. Lưu vào notes cùng với App ID.

Bí mật tuyệt đối

App Secret giống như mật khẩu — KHÔNG share, KHÔNG commit lên Git. Sẽ lưu vào file .env ở Bước 05.

AI-First — UI Facebook đổi liên tục

Facebook for Developers cập nhật giao diện rất thường xuyên. Nếu bạn không thấy nút giống mô tả, screenshot màn hình → paste agent coding + hỏi: "Tôi đang ở Facebook Developer App, muốn tạo App Business / lấy App Secret, bấm đâu?". Đừng đoán mò.

⚠️ Troubleshoot — Bước 02
"Bị block không cho tạo App"

Tài khoản Facebook cá nhân của bạn quá mới hoặc thiếu thông tin. Cần: xác thực số điện thoại, đăng nhập ổn định 2-4 tuần, có ảnh đại diện thật. Nếu account quá mới (<6 tháng), có thể không tạo được — mượn tài khoản người thân/bạn (admin Page chung) để tạo.

"Không thấy option Business khi tạo App"

Bạn đang xem giao diện cũ. Bấm Cancel, vào lại developers.facebook.com/apps, bấm Create App lại — đảm bảo dùng giao diện mới (giao diện trắng, không phải xanh dương cũ).

"Quên App ID/Secret rồi"

Vào lại developers.facebook.com/apps → chọn app vừa tạo → App settings → Basic. Mọi thông tin ở đó.

BƯỚC 03 Lấy Permanent Page Access Token

Đây là phần khó nhất ngày hôm nay. Đọc kỹ, làm chậm. Mục tiêu cuối: có 1 chuỗi token không bao giờ hết hạn, dùng cho code đăng bài lên Page mãi mãi.

AI-First — luật vàng cho Bước 03

Bước này có nhiều thuật ngữ kỹ thuật (token, OAuth, curl, JSON...). Đừng cố hiểu hết — chỉ cần làm đúng thứ tự. Mỗi sub-step:

  1. Đọc hướng dẫn 1 lượt
  2. Có chỗ không hiểu → screenshot + hỏi agent coding "giải thích chỗ này"
  3. Có lệnh curl → paste nguyên lệnh vào agent + bảo "chạy giúp tôi, in output ra"
  4. Lỗi → copy error → paste agent + bảo "lỗi này nghĩa là gì, sửa sao"

Học viên KHÔNG cần là dev. Bạn cần biết bấm "Hỏi AI" đúng lúc.

Tại sao có 4 sub-step nhỏ?
Facebook cố tình thiết kế phức tạp — vì token có quyền đăng bài thay bạn, không thể cho dễ. Bạn phải "đổi" token 3 lần: ngắn → dài → vĩnh viễn. Mỗi lần đổi là 1 lệnh đơn giản, nhưng phải làm đúng thứ tự.

3.1 — Lấy Short-lived User Token (sống 1 giờ)

  1. Vào Graph API Explorer: https://developers.facebook.com/tools/explorer
  2. Góc phải có dropdown Meta App → chọn app bạn vừa tạo (kp3-content-bot).
  3. Phía dưới có nút Add a Permission — thêm 4 permissions sau (gõ tên trong ô tìm rồi tick):
    • pages_manage_posts
    • pages_read_engagement
    • pages_show_list
    • public_profile
  4. Bấm Generate Access Token (nút xanh dương phía trên).
  5. Popup hiện ra → Continue as [Tên bạn] → chọn Page bạn muốn đăng (chỉ tick Page kinh doanh, không tick tất cả) → Continue → tick tất cả permissions → Save.
  6. Bạn được trả về Graph Explorer, ô Access Token bây giờ có 1 chuỗi dài bắt đầu bằng EAA....
Kiểm tra điểm

Bạn copy được 1 chuỗi token bắt đầu bằng EAA, dài ~200 ký tự. Token này chỉ sống 1 giờ — phải đổi thành long-lived ngay ở 3.2.

3.2 — Đổi sang Long-lived User Token (sống 60 ngày)

AI-First — đừng tự gõ terminal

Từ đây tới hết Bước 03, bạn sẽ thấy nhiều lệnh curl. Không cần biết curl là gì. Mở agent coding (Cursor/Antigravity/Claude Code), paste lệnh + bảo: "Chạy lệnh này giúp tôi, giải thích output trả về." Agent sẽ chạy và dịch kết quả cho bạn.

Cách này cũng an toàn hơn — agent biết thay placeholder thế nào, biết debug nếu lỗi, biết gợi ý nếu bạn paste sai chỗ.

Mở terminal trên máy (Terminal trên Mac, PowerShell/CMD trên Windows), paste lệnh sau — thay 3 chỗ in đậm bằng giá trị thật của bạn:

curl -G "https://graph.facebook.com/v18.0/oauth/access_token" \
  -d "grant_type=fb_exchange_token" \
  -d "client_id=APP_ID_CỦA_BẠN" \
  -d "client_secret=APP_SECRET_CỦA_BẠN" \
  -d "fb_exchange_token=SHORT_LIVED_TOKEN_TỪ_3.1"

Trả về JSON kiểu:

Output mong đợi
{"access_token":"EAAxxxxxx...","token_type":"bearer","expires_in":5183944}

Copy chuỗi access_token này — đây là Long-lived User Token sống 60 ngày.

3.3 — Đổi sang Permanent Page Token (KHÔNG hết hạn)

Đây là token cuối cùng — vĩnh viễn. Cần biết User ID của bạn trên Facebook trước.

Lấy User ID:

curl "https://graph.facebook.com/v18.0/me?access_token=LONG_LIVED_TOKEN_TỪ_3.2"
Output
{"name":"Tên Bạn","id":"10223611603830114"}

Copy id — đây là User ID (chuỗi số dài).

Lấy Permanent Page Token:

curl "https://graph.facebook.com/v18.0/USER_ID/accounts?access_token=LONG_LIVED_TOKEN"
Output
{ "data": [ { "access_token": "EAAxxxxxxx...", ← ĐÂY là Permanent Page Token "category": "Brand", "name": "Tên Page của bạn", "id": "123456789012345" ← ĐÂY là Page ID } ] }

Copy 2 chuỗi:

  • access_token — đây là Permanent Page Token. KHÔNG bao giờ hết hạn.
  • id — đây là Page ID của bạn. Cũng cần lưu.

3.4 — Kiểm tra token vĩnh viễn

Để chắc chắn token thật sự không hết hạn, vào Access Token Debugger: https://developers.facebook.com/tools/debug/accesstoken/

  1. Paste Permanent Page Token vào ô.
  2. Bấm Debug.
  3. Tìm dòng Expires — phải là Never.
Kiểm tra điểm — quan trọng nhất

Nếu thấy Expires: Never — bạn đã thành công. Token này lưu vào file .env là dùng vĩnh viễn.

Nếu thấy thời gian hết hạn — bạn đã làm sai 1 bước. Quay lại 3.2 hoặc 3.3, làm lại cẩn thận.

⚠️ Troubleshoot — Bước 03
"Token short-lived ở 3.1 không có chuỗi EAA..."

Bạn quên thêm permissions hoặc chưa "Continue as Tên bạn" + chọn Page. Bấm Generate Access Token lại, đọc kỹ popup, tick đầy đủ.

"3.2 trả về error 'Invalid OAuth access token'"

Token short-lived đã hết hạn (1 giờ trôi nhanh khi đang debug). Quay lại 3.1, lấy token mới, làm 3.2 ngay trong vòng 5 phút.

"3.3 trả về data trống []"

Bạn chưa phải admin của Page nào, hoặc Page đó là Profile cá nhân (không phải Business Page). Tạo 1 Business Page trên Facebook trước, rồi quay lại 3.1.

"Permanent token vẫn hiện expires"

Có 2 lý do: (1) Bạn đổi short-lived → permanent, không qua long-lived. Phải làm đủ 3 bước. (2) Account FB chưa đủ tin cậy. Thử account khác, hoặc đợi vài ngày.

"Tôi không quen terminal, làm sao?"

Có 2 cách thay thế:

  • Postman (app GUI free): tạo GET request, paste URL với params.
  • Hỏi agent coding: paste lệnh curl, agent chạy giúp.
Phần Hai
Vibe Code Skill 2 Mode

BƯỚC 04 Hiểu skill 2 mode hôm nay

Skill bạn vibe hôm nay là Cấp 3 (theo phân loại Ngày 16) — có scripts/ chạy code. Skill này có 2 chế độ hoạt động:

Mode 1 · Content Free
Đăng đều mỗi ngày, không spam

Agent gen 3 ý tưởng bài đăng organic (chia sẻ giá trị, không hard sell). Bạn chọn 1, agent tạo full content: ảnh đẹp + caption hoàn chỉnh có hook + body + CTA mềm — đăng tự động lên Page. Lịch hẹn 9h sáng mỗi ngày kích hoạt.

Trigger: "Tạo content cho ngày mai", "Gen bài Page", "Content free"

Mode 2 · Creative Ads
Sản xuất hàng loạt khi chạy chiến dịch

Agent gen 3 bộ creative hoàn chỉnh — mỗi bộ gồm 1 ảnh ads (visual mạnh) + 1 ad copy (hook + USP + CTA rõ ràng). Bạn copy paste cả 2 phần vào Ads Manager để chạy. Không tự đăng — vì ads cần bạn duyệt budget, target.

Trigger: "Tạo creative ads", "Gen ads cho sản phẩm X", "Cần 3 bộ creative" (mỗi bộ = ảnh + copy)

Tại sao 1 skill 2 mode (không phải 2 skill riêng)?

Vì cùng dùng chung 1 hạ tầng — cùng OpenAI API, cùng kiến thức brand voice, cùng Python script. Khác biệt chỉ ở prompt engineering (cách viết prompt cho GPT Image) và action cuối (đăng vs để học viên tự dùng). Tách 2 skill = duplicate nhiều code.

BƯỚC 05 Vibe code skill bằng agent coding

Mở agent coding (Cursor / Antigravity / Claude Code) trong thư mục my-skills/ bạn đã tạo ở Ngày 16. Paste prompt:

Prompt · Vibe skill tao-creative-fb (AI-First)

⚠️ Troubleshoot — Bước 06
"Lỗi 401 từ OpenAI: Invalid API key"

Key sai hoặc thiếu. Kiểm tra: file .env có OPENAI_API_KEY=sk-proj-... đúng định dạng không. Không có dấu cách trước/sau dấu =. Không có nháy "" bao quanh.

"Lỗi 429 từ OpenAI: rate limit"

Free tier có giới hạn ~3 request/phút. Đợi 1 phút, chạy lại. Hoặc nâng tier (nạp thêm tiền vào account).

"Ảnh gen ra xấu/không đúng concept"

Mở assets/image-prompt-templates.md — đây là nơi điều chỉnh prompt cho GPT Image. Bảo agent: "Cải thiện template để ảnh có style [tả style mong muốn]". Chạy lại.

"Caption gen ra quá AI-generic, không giống tôi"

Mở assets/caption-templates.md — đây là template cho văn bản. Bảo agent: "Đọc thêm brand_voice trong brain.db, viết lại template để giống cách tôi nói". Có thể paste 2-3 bài đăng cũ của bạn để agent học giọng.

"Lỗi từ Facebook: (#200) Permissions error"

Token thiếu permissions pages_manage_posts. Quay lại Bước 03.1, generate token mới với đầy đủ 4 permissions.

"Lỗi (#190) Token expired"

Bạn dùng nhầm short-lived hoặc long-lived token (60 ngày). Phải dùng Permanent Page Token từ 3.3. Vào developers.facebook.com/tools/debug/accesstoken kiểm tra Expires phải là Never.

"Bài đăng lên Page nhưng không có ảnh"

Code chỉ post text, chưa upload ảnh. Hỏi agent: "Sửa post_facebook.py để upload ảnh kèm caption qua endpoint /{page-id}/photos".

"Bài đăng thành công nhưng người ngoài / khách không thấy"

App đang ở chế độ Development — Facebook chỉ cho admin/mod của Page thấy bài, người ngoài không thấy (dù bài vẫn hiện icon 🌐 trông như public). Sửa: Dashboard App → gạt công tắc Development → Live ở thanh trên cùng (cần điền Privacy Policy URL + chọn Category ở App settings → Basic mới gạt được). Permission để mức Standard Access là đủ, không cần App Review. Kiểm tra lại bằng cửa sổ ẩn danh hoặc nhờ người không có vai trò trên Page xem.

BƯỚC 08 Khi nào dùng Mode Ads

Mode 1 (content free) chạy auto. Mode 2 (creative ads) không chạy auto — chỉ kích hoạt khi bạn cần. Tại sao?

  • Ads tốn tiền thật → cần bạn duyệt từng cái
  • Mỗi chiến dịch khác nhau (target, USP, ưu đãi) → không gen đại trà được
  • Bạn vẫn paste vào Ads Manager → Facebook lo việc chạy/tối ưu

Khi nào kích hoạt

  • Sắp launch sản phẩm mới → cần 5-10 mẫu creative để test
  • Ads hiện tại bị fatigue (CTR giảm) → cần creative mới
  • Có sự kiện/khuyến mãi → cần ads cho sự kiện đó

Cách dùng

Trong Telegram, gõ:

Tạo creative ads cho [sản phẩm]
- Giá: [giá]
- USP: [3 điểm khác biệt nổi bật]
- Audience: [mô tả khách hàng mục tiêu]
- Mục tiêu: [lead gen / sales / awareness]

Cần 3 bộ creative — mỗi bộ gồm:
- 1 ảnh ads (visual mạnh, có khoảng trống cho text)
- 1 ad copy hoàn chỉnh (hook + USP + CTA)

3 bộ phải khác angle nhau:
- Bộ 1: pain point (khoét nỗi đau)
- Bộ 2: solution (giải pháp + lợi ích)
- Bộ 3: social proof (testimonial / kết quả thực)

Agent gen → gửi qua Telegram. Bạn:

  1. Tải cả 3 ảnh về máy (mỗi ảnh đi với 1 copy tương ứng)
  2. Copy cả 3 đoạn ad copy (mỗi đoạn ~80-150 từ)
  3. Mở Facebook Ads Manager → tạo 3 ads riêng — mỗi ad upload 1 ảnh + paste 1 copy tương ứng
  4. Set budget, audience, chạy. Sau 3 ngày xem ad nào CTR cao nhất → scale ngân sách lên cái đó.
Mẹo nâng cao

Yêu cầu agent gen 3 angle khác nhau: pain point + solution + social proof. Test cả 3, bài nào CTR cao nhất sau 3 ngày → scale ngân sách lên cái đó.

Nếu Dashboard không có UI upload skill, bảo agent coding: "SSH lên VPS chạy goClaw, copy thư mục skill vào folder skills của agent, hướng dẫn từng bước."

BƯỚC 10 Quan sát + Tinh chỉnh trong 7 ngày tới

Khác với các ngày trước, hôm nay không có bài đăng khoe. Lý do: chất lượng skill chỉ thấy được sau khi nó chạy thật vài ngày. Nhiệm vụ của bạn 7 ngày tới:

  1. Mỗi sáng 9h: Telegram bật, agent gửi 3 ý tưởng. Bạn chọn, duyệt, post.
  2. Sau 3-4 ngày: review lại các bài đã đăng. Câu hỏi tự hỏi:
    • Tone đã đúng giọng business chưa?
    • Ảnh có style nhất quán không?
    • Caption có quá AI-generic không?
  3. Tinh chỉnh template: nếu thấy bài chưa đúng, mở agent coding, sửa template trong assets/image-prompt-templates.md (cho ảnh) hoặc assets/caption-templates.md (cho văn bản). AI sẽ làm giúp — chỉ cần nói "Sửa template để bài đăng có style [mô tả]".
  4. Lưu lại moments đáng khoe: screenshot Page khi có bài auto-post đẹp, screenshot Telegram lúc bạn duyệt — để dành cho Demo Day Ngày 21.
Demo Day Ngày 21 đang chờ

Cuối thử thách (Ngày 21), sẽ có buổi meeting trình bày hệ thống bạn đã build. Lúc đó bạn sẽ show cả Page Facebook auto-post hằng ngày như 1 phần của hệ thống — đây là moment khoe ấn tượng nhất, không phải đăng bài rời rạc giữa chừng.

Vậy nên hôm nay tập trung làm cho skill chạy ngon — quà sẽ đến ở Ngày 21.

Mẹo AI-First quan trọng

Đừng tự tay sửa template hay code — luôn đưa cho agent coding làm. Bạn chỉ nói mong muốn + đưa ví dụ bài đăng đẹp/xấu. Agent sẽ refactor template hoặc viết lại prompt cho bạn.

Đây là pattern xuyên suốt 21 ngày: bạn là người chỉ đạo, AI là người làm. Càng dùng nhiều, càng quen.
