Thành Vũ Khí AI

Bộ skills 2 triệu tôi tặng các bạn ở Ngày 15 — hôm nay bạn sẽ tự tay tạo những skill còn xịn hơn thế, chính xác cho business của riêng bạn. Skill là cách để agent thay được những người làm chuyên môn cao — lập trình viên, video editor, copywriter, kế toán...

Mục tiêu cuối ngày

Cuối hôm nay bạn có:

Bộ skills 2tr tôi tặng ở Ngày 15 — đã được cài đặt và dùng được trong agent coding của bạn + trên goClaw Ít nhất 3 skill do chính bạn tạo, gắn với business của bạn (viết bài Facebook đúng giọng, soạn proposal đúng template, làm báo cáo doanh thu... — tùy bạn chọn) Hiểu rõ 5 cấp độ skill, biết khi nào dùng cấp nào — bằng cách làm bài trắc nghiệm 10 câu
Một câu chuyện thật

Bạn từng phải trả 10 triệu cho 1 video editor để cắt 30 video reels mỗi tháng? Hoặc 15 triệu cho 1 lập trình viên junior để code 1 tính năng mới mỗi tuần? Hoặc 5 triệu cho 1 copywriter để viết content theo brand voice của bạn?

Skill cấp cao = cho agent khả năng làm những việc đó thay con người. Bạn ngủ — agent edit video, code, viết bài. Không phải tương lai xa — đây là cái bạn build được trong vài giờ hôm nay.

Tinh thần ngày hôm nay

Đây là ngày chuyển hóa quan trọng nhất chuỗi. Trước hôm nay bạn nhận skill từ tôi. Sau hôm nay bạn tự tạo skill — đồng nghĩa bạn không còn phụ thuộc vào ai nữa, business của bạn cần gì tự làm được.

SOP hôm nay ép bạn dùng agent coding hỏi — không phải đọc lý thuyết suông. Đây mới là cách hiểu sâu.

BƯỚC 01 Giải nén bộ skills bạn nhận được

Nhắc lại

Hoàn thành Ngày 15, tôi đã gửi bạn 1 bộ skills xịn xò trị giá 2 triệu — gồm các skill viết content, soạn proposal, phân tích đối thủ... Hôm nay, trước khi học cách tự tạo skill mới, hãy cài đặt và dùng được bộ skills cũ này trước.

Vì sao? Vì bạn cần thấy nó hoạt động để hiểu skill là gì trên thực tế. Lý thuyết đọc 100 lần không bằng dùng thử 1 lần.

  1. Tải bộ skills tôi đã gửi bạn (file .zip qua email/Telegram khi bạn hoàn thành Ngày 15).
  2. Giải nén ra một thư mục, đặt tên dễ nhớ — ví dụ kp3-skills/. Bên trong sẽ có nhiều thư mục con, mỗi thư mục là 1 skill (kebab-case như viet-bai-facebook/, soan-proposal/...).
  3. Mở thử 1 thư mục — bạn sẽ thấy file SKILL.md bên trong. Đây chính là "trái tim" của skill.
Chưa nhận được bộ skills?

Có thể bạn chưa hoàn thành Ngày 15 đúng cách (cần tag Facebook Dương Trọng Nghĩa). Quay lại Ngày 15, hoàn thành đúng quy trình, bộ skills sẽ được gửi cho bạn.

Trong lúc chờ, bạn vẫn có thể bỏ qua Phần Một và làm Phần Hai trở đi — học lý thuyết + tự tạo skill mới của bạn.

BƯỚC 02 Cài skills vào agent coding của bạn

Có 2 cách dùng tùy bạn đang dùng agent coding nào:

Cách A — Dùng "@filename trick" (cho mọi agent coding)

Đây là cách đơn giản nhất, hoạt động trên mọi agent coding (Cursor, Antigravity, Windsurf, Continue, Claude Code...):

  1. Copy thư mục kp3-skills/ vào trong dự án bạn đang làm việc (ví dụ thư mục website của bạn).
  2. Mở agent coding, trong khung chat gõ ký tự @ → một popup hiện ra → tìm và chọn file SKILL.md của skill bạn muốn dùng (ví dụ kp3-skills/viet-bai-facebook/SKILL.md).
  3. File được tạm thời inject vào ngữ cảnh hội thoại. Agent đã đọc skill.
  4. Sau đó gõ yêu cầu thật, ví dụ: "Viết bài Facebook về khoá học X giá 2tr, có ưu đãi cuối tuần."
  5. Agent đọc skill → làm theo template/instructions → ra output đúng format.
@filename trick là gì?
Mỗi agent coding đều có cách tạm thời chèn 1 file vào ngữ cảnh hội thoại. Cursor/Windsurf/Antigravity gõ @ hoặc #. Đây là cách dùng skill tạm thời cho 1 cuộc trò chuyện — không cài permanent, lúc nào cần thì gọi.

Cách B — Cài permanent vào Claude Code (chỉ cho ai có Claude Code)

Nếu bạn dùng Claude Code, có thể cài skill permanent để Claude tự kích hoạt khi cần, không phải gọi @ mỗi lần:

  1. Mở terminal, copy bộ skills vào thư mục skills của Claude Code: 
    cp -r kp3-skills/* ~/.claude/skills/
  2. Khởi động lại Claude Code (đóng và mở lại terminal).
  3. Từ giờ, mỗi khi bạn nói câu phù hợp với skill (ví dụ "viết bài Facebook"), Claude tự nhận ra, tự đọc skill, tự làm theo. Không cần gọi @.
Không có Claude Code?

Không sao. Cách A (@filename) hoạt động trên hầu hết agent coding khác. Bạn không bỏ lỡ gì cả — chỉ là thao tác mỗi lần dùng thêm 1 bước (gõ @).

Khi quen với skills rồi, bạn có thể mua Claude Code (rẻ hơn nhiều so với thuê 1 nhân viên).

BƯỚC 03 Upload skills lên Claude.ai (tùy chọn)

Nếu bạn có tài khoản Claude.ai (Pro hoặc Free), bạn có thể upload skills lên đó để dùng trực tiếp trong giao diện chat của Claude — không cần agent coding:

  1. Mở claude.ai → đăng nhập.
  2. Vào Settings (góc trên phải) → Capabilities → Skills.
  3. Bấm Upload Skill → chọn thư mục skill (nén thành .zip trước nếu yêu cầu).
  4. Bật Enabled cho skill đó.
  5. Quay về cuộc chat — gõ yêu cầu phù hợp, Claude tự kích hoạt skill.
Không có Claude.ai?

Bỏ qua bước này — Cách A (@filename trong agent coding) đã đủ rồi. Hoặc đến Phần Bốn bạn sẽ upload skills lên goClaw để dùng qua Telegram.

BƯỚC 04 Dùng thử ít nhất 1 skill

Đừng bỏ qua bước này. Trải nghiệm thật mới hiểu skill là gì.

Chọn 1 skill trong bộ kp3-skills/ phù hợp với business của bạn (ví dụ viet-bai-facebook/). Mở skill ra (đọc nhanh SKILL.md để hiểu nó làm gì), rồi:

  1. Trong agent coding, gõ @ → chọn SKILL.md của skill đó.
  2. Gõ yêu cầu thật từ business của bạn — không phải ví dụ giả. Ví dụ: "Viết bài Facebook giới thiệu khoá [tên khoá thật của bạn], giá [giá thật], điểm khác biệt là [USP thật]."
  3. Đọc kỹ output — agent có ra đúng format không? Tone có đúng không?
  4. Nếu output chưa đúng giọng bạn — đó là dấu hiệu skill cần điều chỉnh. Đến Phần Ba bạn sẽ học cách sửa skill.
Khoảnh khắc bạn cần cảm nhận

Khi bạn paste 1 yêu cầu ngắn 1 câu vào agent coding, kèm @SKILL.md, agent ra cho bạn 1 bài đăng Facebook hoàn chỉnh đúng format trong 10 giây — đó chính là giá trị của skill. Trước đây bạn phải mô tả lại context mỗi lần. Giờ skill làm thay.

Nhân giá trị đó với 30 lần/tháng — đó là tại sao bộ skills này đáng 2tr.

BƯỚC 05 Đọc ebook trước

Tôi đã gửi bạn file ebook "The Complete Guide to Building Skills for Claude" của Anthropic. Đây là tài liệu chính thức.

Bạn không cần đọc hết. Chỉ cần đọc 3 chương:

  • Chapter 1 — Fundamentals: Skill là gì, cấu trúc thư mục, progressive disclosure. ~3 trang.
  • Chapter 2 — Planning and Design: Cách viết SKILL.md + YAML frontmatter. ~6 trang.
  • Chapter 5 — Patterns and Troubleshooting (chỉ phần Patterns đầu): 5 patterns phổ biến. ~5 trang.

Tổng: ~14 trang, đọc 20-30 phút. Đọc bằng tiếng Anh nhưng có agent coding bên cạnh — không hiểu chỗ nào, paste vào hỏi.

Hiểu skill kiểu đơn giản nhất
Skill = 1 thư mục có file SKILL.md. File này dạy agent cách làm 1 việc cụ thể. Khi agent thấy task tương tự, nó tự đọc skill và làm theo.

BƯỚC 06 Hiểu 5 cấp độ skill

Skill có thể đơn giản (chỉ 1 file MD) hoặc phức tạp (nhiều thành phần). Anthropic chia thành 5 cấp độ — biết cấp nào phù hợp việc nào là kỹ năng quan trọng nhất hôm nay:

Cấp 1 · Skill Đơn
Chỉ 1 file SKILL.md
my-skill/ └── SKILL.md
Khi nào dùng: Khi skill chỉ là hướng dẫn cách agent xử lý task — không cần data ngoài, không cần format chuẩn cứng. Output là văn bản tự do.

Thay được: junior CS, junior copywriter, content creator

Cấp 2 · Skill + Khuôn Mẫu
SKILL.md + assets/ chứa template
my-skill/ ├── SKILL.md └── assets/ └── template.docx
Khi nào dùng: Khi output cần đúng format chuẩn mỗi lần — slide, hợp đồng, proposal, caption có hook+body+CTA cố định.

Thay được: graphic designer (làm template), proposal writer, SEO writer

Cấp 3 · Skill + Máy Tự Động
SKILL.md + scripts/ chứa code chạy được
my-skill/ ├── SKILL.md └── scripts/ └── process.py
Khi nào dùng: Khi cần xử lý dữ liệu thực tế — đếm, tính toán, parse file, validate, edit video, render ảnh. Code chính xác hơn ngôn ngữ.

Thay được: lập trình viên junior, data analyst, video editor (FFmpeg), SEO checker

Cấp 4 · Skill + Sổ Tay Chuyên Môn
SKILL.md + references/ chứa kiến thức sâu
my-skill/ ├── SKILL.md └── references/ └── deep-knowledge.md
Khi nào dùng: Khi skill cần kiến thức chuyên sâu — luật, tài chính, y khoa, kế toán. Agent tham chiếu khi cần (không load hết vào context cho đỡ tốn chi phí AI).

Thay được: tư vấn pháp lý, tư vấn tài chính, kế toán, chuyên gia chuyên môn

Cấp 5 · Skill Đầy Đủ
SKILL.md + scripts/ + assets/ + references/
my-skill/ ├── SKILL.md ├── scripts/ ├── assets/ └── references/
Khi nào dùng: Cho task phức hợp cao — vừa cần xử lý data, vừa cần format chuẩn, vừa cần kiến thức sâu. Ví dụ: build báo cáo BI hàng tuần.

Thay được: senior all-round (technical writer + designer + dev junior)

Quy tắc vàng khi chọn cấp

Bắt đầu cấp thấp nhất khả thi. Skill đầu tiên của bạn nên Cấp 1 hoặc Cấp 2 — dễ làm, dễ test, ra kết quả nhanh. Sau quen rồi, lên Cấp 3-5 cho task phức tạp.

Đừng over-engineer skill đầu tiên — không ai làm được skill cấp 5 ngay lần đầu.

BƯỚC 07 Trắc nghiệm 10 câu — bắt buộc dùng AI để trả lời

Cách làm bài

Mỗi câu là 1 tình huống thật. Bạn phải đoán: cấp skill nào phù hợp nhất?

Quy tắc bắt buộc: với mỗi câu, mở agent coding của bạn (Cursor / Antigravity / Claude Code) và bấm nút "Hỏi AI" — nó copy sẵn prompt, bạn dán vào agent, agent giải thích. Sau đó quay lại đây chọn đáp án.

Đừng đoán mò. Mục tiêu là hiểu sâu, không phải đúng nhanh. Làm đủ 10 câu, mỗi câu sẽ có giải thích chi tiết sau khi bạn chọn.

0 / 10 câu đã hoàn thành
Câu 01 / 10

Bạn muốn agent trả lời câu hỏi thường gặp của khách hàng trên Telegram, đúng tone business của bạn. FAQ có sẵn 20 câu, mỗi câu trả lời ngắn 2-3 dòng.

Chưa biết? Bấm "Hỏi AI" → dán vào Cursor/Antigravity → đọc giải thích → quay lại chọn đáp án.
A. Cấp 1 — Chỉ SKILL.md với danh sách FAQ + cách phản hồi
B. Cấp 2 — SKILL.md + assets/faq.md (tách FAQ ra file riêng)
C. Cấp 3 — Cần script Python để xử lý câu hỏi
D. Cấp 4 — Cần references chuyên sâu

Đáp án: A — Cấp 1. Trả lời FAQ là task đơn giản, chỉ cần hướng dẫn thuần văn bản: "Khi khách hỏi câu A → trả lời theo cách B, giọng theo brand voice". Không cần script (không có data cần xử lý), không cần template (output là tin nhắn tự do), không cần references chuyên sâu (FAQ ngắn gọn).

Vì sao không phải Cấp 2? assets/ dùng cho template OUTPUT — file mẫu mà agent fill nội dung vào (ví dụ proposal-template.docx, caption-template.md). Nó KHÔNG phải nơi chứa knowledge/FAQ. 20 câu × 2-3 dòng ≈ 60 dòng → fit thoải mái trong SKILL.md, không cần tách ra. Nếu AI trả lời "Cấp 2" là AI hiểu sai vai trò của assets/.

Câu 02 / 10

Bạn muốn agent edit video tự động — cắt clip dài thành 5 reels ngắn, thêm caption animation, ghép nhạc nền.

Chưa biết? Bấm "Hỏi AI" → dán vào agent coding → đọc giải thích.
A. Cấp 1 — SKILL.md với hướng dẫn cách edit
B. Cấp 2 — SKILL.md + template caption mẫu
C. Cấp 3 — SKILL.md + scripts/ chứa FFmpeg/Python xử lý video
D. Cấp 4 — SKILL.md + references/ về nguyên tắc montage

Đáp án: C — Cấp 3. Edit video cần xử lý dữ liệu thực tế (file video). Không thể chỉ "mô tả bằng lời" — phải có script Python/Bash gọi FFmpeg để cắt, ghép, render. Đây là điển hình của task thay được video editor (~10tr/tháng).

Câu 03 / 10

Bạn muốn agent soạn proposal/báo giá cho khách hàng mới — phải đúng template thương hiệu của bạn (logo, màu, layout cố định).

Chưa biết? Bấm "Hỏi AI" để nó giải thích cho bạn.
A. Cấp 1 — Chỉ SKILL.md mô tả nội dung proposal
B. Cấp 2 — SKILL.md + assets/proposal-template.docx có sẵn layout
C. Cấp 3 — Cần script Python tạo file Word
D. Cấp 4 — Cần references về kỹ thuật bán hàng

Đáp án: B — Cấp 2. Output cần đúng format brand cố định mỗi lần (logo, màu, layout). Cách tốt nhất: cho agent file template có sẵn, agent chỉ cần fill vào. Không cần script (không xử lý data), không cần references (kiến thức bán hàng đã có thể nằm trong SKILL.md).

Câu 04 / 10

Bạn muốn agent code 1 tính năng mới vào website của bạn — ví dụ thêm trang "Khách hàng nói gì". Task gồm: viết code frontend, viết code backend, test, commit, deploy.

Đây là task khó nhất bài. Bấm "Hỏi AI" để hiểu rõ.
A. Cấp 1 — SKILL.md mô tả task + brand
B. Cấp 2 — Chỉ template code mẫu
C. Cấp 4 — Chỉ references về cách code
D. Cấp 5 — SKILL.md + scripts (test, deploy) + assets (code template) + references (best practices)

Đáp án: D — Cấp 5. Tasks code phức tạp = nhiều bước (write code, test, deploy) → cần đủ cả ba: scripts để tự test/deploy, assets chứa code template chuẩn của bạn, references chứa best practices của codebase. Đây là skill thay được junior dev (~15tr/tháng).

Câu 05 / 10

Bạn muốn agent soạn hợp đồng cho khách thuê dịch vụ — phải đúng luật VN, có điều khoản chuẩn về thanh toán, sở hữu trí tuệ, NDA, giải quyết tranh chấp.

Chưa biết? Bấm "Hỏi AI" để nó giải thích cho bạn.
A. Cấp 1 — Chỉ hướng dẫn thuần
B. Cấp 2 — Chỉ template hợp đồng
C. Cấp 4 — SKILL.md + references/ chứa các điều khoản pháp lý chuẩn
D. Cấp 3 — Cần script tạo file PDF

Đáp án: C — Cấp 4. Soạn hợp đồng cần kiến thức chuyên sâu về luật. Mỗi điều khoản (thanh toán, IP, NDA, tranh chấp) là 1 chuyên đề riêng — không thể nhét hết vào SKILL.md. Cách đúng: references/ chứa từng file MD cho mỗi loại điều khoản, agent đọc khi cần. Đây là skill thay được tư vấn pháp lý (~5-10tr/lần).

Câu 06 / 10

Bạn muốn agent làm báo cáo doanh thu cuối tháng — đọc data thật từ database (đơn hàng, khách hàng), tính toán doanh thu/lãi/khách mới, vẽ biểu đồ, xuất ra slide PowerPoint theo template công ty.

Câu này tổng hợp nhiều thứ. Bấm "Hỏi AI" để hiểu rõ.
A. Cấp 1 — Hướng dẫn thuần
B. Cấp 2 — Chỉ template slide
C. Cấp 3 — Chỉ script đọc database
D. Cấp 5 — Đầy đủ: scripts (đọc DB + tính số), assets (template slide), references (cách diễn giải số liệu)

Đáp án: D — Cấp 5. Task này phức hợp 3 thứ: (1) đọc + tính data → cần scripts, (2) format slide đúng template công ty → cần assets, (3) diễn giải số liệu kiểu chuyên nghiệp → cần references. Skill này thay được data analyst (~20tr/tháng).

Câu 07 / 10

Bạn muốn agent viết caption Instagram cho mỗi sản phẩm bạn bán — caption có cấu trúc cố định: Hook (câu gây chú ý) + Body (mô tả sản phẩm) + CTA (kêu gọi mua).

Chưa biết? Bấm "Hỏi AI" để nó giải thích cho bạn.
A. Cấp 1 — Chỉ SKILL.md mô tả cách viết caption
B. Cấp 2 — SKILL.md + assets/caption-template.md có Hook/Body/CTA mẫu
C. Cấp 3 — Cần script đọc thông tin sản phẩm
D. Cấp 4 — Cần references về marketing

Đáp án: B — Cấp 2. Caption cần cấu trúc cố định (Hook + Body + CTA) → cần template làm sẵn cho agent fill. Tương tự câu 03 (proposal): khi output có format chuẩn, dùng Cấp 2 với template trong assets/.

Câu 08 / 10

Bạn muốn agent phân tích đối thủ cạnh tranh — bạn cho tên đối thủ, agent vào website của họ, đọc nội dung, đánh giá điểm mạnh/yếu, viết báo cáo dạng văn bản.

Chưa biết? Bấm "Hỏi AI" để nó giải thích cho bạn.
A. Cấp 1 — Chỉ SKILL.md với hướng dẫn phân tích theo framework SWOT
B. Cấp 2 — Cần template báo cáo
C. Cấp 3 — Cần script crawl website
D. Cấp 5 — Cần đầy đủ

Đáp án: A — Cấp 1. Phân tích đối thủ là task research + suy luận — agent dùng tool có sẵn (web search/fetch) tự đọc website, không cần script crawl đặc biệt. Output là văn bản tự do, không cần template cứng. Skill chỉ cần dạy framework phân tích (ví dụ SWOT).

Câu 09 / 10

Bạn muốn agent tạo poster quảng cáo tự động — bạn nhập thông tin sự kiện (tên, ngày, địa điểm), agent tạo ra ảnh poster có sẵn typography đẹp, màu sắc đúng brand, logo công ty.

Câu này nâng cao. Bấm "Hỏi AI" để hiểu rõ.
A. Cấp 1 — Chỉ hướng dẫn thiết kế
B. Cấp 2 — Chỉ assets/ chứa logo + font
C. Cấp 3 — SKILL.md + scripts/ (Python với Pillow/SVG để render ảnh) + assets/ (logo, font, màu)
D. Cấp 4 — Cần references về thiết kế

Đáp án: C — Cấp 3 (kết hợp với assets). Tạo ảnh là task cần code chạy được (Python với thư viện Pillow, hoặc SVG/Canvas). Đồng thời cần assets (logo, font). Đây là Cấp 3 mở rộng có cả assets — skill thay được junior designer (~10-15tr/tháng).

Câu cuối! Bấm "Hỏi AI" để hiểu rõ.

Quy tắc khi viết description trong YAML

Theo ebook (Chapter 2), description là phần quan trọng nhất của skill — nó quyết định khi nào skill kích hoạt. Description tốt phải gồm:

  • Cái gì (What it does)
  • Khi nào dùng (When to use it) + các cụm từ người dùng có thể nói
  • Dưới 1024 ký tự

Ví dụ tốt: "Viết bài Facebook theo brand voice. Dùng khi user nói 'viết bài cho sản phẩm X', 'làm caption Facebook', 'viết content quảng cáo'."

BƯỚC 11 Test skill ngay trong agent coding

Sau khi agent đã viết xong, test ngay tại agent coding bằng "@filename trick" — đây là cách hầu hết agent coding (Cursor, Antigravity, Windsurf) đều support:

  1. Trong khung chat của agent coding, gõ @ → một popup hiện ra → chọn file SKILL.md bạn vừa tạo. File được inject vào context.
  2. Sau đó gõ yêu cầu thật, ví dụ: "Viết bài Facebook về khoá học Vibe Coding của tôi, giá 2tr, ưu đãi cuối tuần."
  3. Agent sẽ đọc skill, làm theo template/instructions trong skill, ra bài đăng.
  4. Kiểm tra: output có đúng format bạn mong muốn không? Tone có đúng không? Nếu không đúng → quay lại sửa SKILL.md, test lại.
@filename trick là gì?
Mỗi agent coding đều có cách tạm thời chèn 1 file vào ngữ cảnh hội thoại. Cursor/Windsurf gõ @, Antigravity gõ # hoặc kéo thả file. Đây là cách nhanh nhất để test skill trước khi upload lên goClaw.
Skill không hoạt động đúng?

3 lỗi phổ biến (theo ebook Chapter 5):

  1. Skill không kích hoạt: description thiếu trigger phrase. Sửa: thêm các cụm người dùng thường nói.
  2. Skill kích hoạt sai lúc: description quá rộng. Sửa: thêm "Do NOT use for [...]" để hạn chế.
  3. Output không đúng template: SKILL.md instructions chưa rõ. Sửa: thêm Examples cụ thể, càng concrete càng tốt.

BƯỚC 12 Upload skill lên goClaw

Skill bạn vừa tạo dùng được trong agent coding (qua @filename). Giờ đưa lên goClaw để agent Telegram của bạn dùng được — bất cứ khi nào bạn nhắn yêu cầu phù hợp, nó tự kích hoạt skill.

  1. Mở https://app.yourdomain.com → đăng nhập goClaw Dashboard.
  2. Sidebar → Capabilities → Skills.
  3. Bấm Upload Skill → chọn thư mục skill bạn vừa tạo (nén thành .zip nếu cần).
  4. Bật Enabled cho skill này.
  5. Vào agent detail (agent Telegram của bạn) → tab Skills → tick để cho agent này dùng skill vừa upload.
Không thấy Skills trong sidebar goClaw?

Tuỳ phiên bản, có thể nằm ở: Capabilities → Skills, hoặc Core → Skills, hoặc cần upload qua filesystem sync (giống Knowledge Vault ở Ngày 15).

Chụp màn hình → paste vào Claude → "tôi muốn upload skill cho goClaw, bấm đâu?".
